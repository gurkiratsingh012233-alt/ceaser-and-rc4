Substitution Cipher

Caesar Cipher

Encrypt messages by shifting the alphabet by a fixed number of positions.

Introduction

What is it?

A substitution cipher where each letter in the plaintext is replaced by a letter some fixed number of positions down the alphabet.

History

Named after Julius Caesar, who used it with a shift of 3 to protect messages of military significance.

Theory

Encryption & Decryption Logic

  1. Choose a shift value (Key) between 1 and 25.
  2. For Encryption: Add the key to the letter's alphabetical index. Wrap around to 'A' if you pass 'Z'.
  3. For Decryption: Subtract the key from the ciphertext letter.

Solved Example (Shift = 3)

Plaintext: HELLO

H + 3 = K
E + 3 = H
L + 3 = O
L + 3 = O
O + 3 = R

Ciphertext: KHOOR

Stream Cipher

RC4 Cipher

Generate a pseudorandom keystream and XOR it with your data.

Introduction

What is it?

A software-based stream cipher that generates a continuous stream of pseudorandom bytes, which are XORed with the plaintext.

History

Designed by Ron Rivest in 1987 for RSA Security. It was famously used in early Wi-Fi security (WEP/WPA).

Theory

1. Key-Scheduling Algorithm (KSA)

RC4 initializes a 256-byte state array (S) from 0 to 255. It then scrambles this array using the secret key to ensure diffusion.

2. Pseudo-Random Generation Algorithm (PRGA)

For each byte of plaintext, the algorithm modifies the state array and outputs a single byte of the keystream (K).

3. XOR Operation

The plaintext byte is XORed with the keystream byte to produce the ciphertext. Because XOR is reversible, decryption uses the exact same process.
